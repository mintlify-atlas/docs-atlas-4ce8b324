Compose Performance Skills ships 26 `SKILL.md` files organized under nested category folders (`<category>/<slug>/SKILL.md`). Agent runtimes expect a flat layout — one folder per skill, directly inside the skills directory. The bundled `install-skills.sh` script handles the translation: it clones the repository once to a stable location and symlinks each individual skill folder into the flat layout your agent expects. Pick the tab for your runtime below.

<Tabs>
  <Tab title="Claude Code">

Claude Code loads skills from `~/.claude/skills/<slug>/SKILL.md`. The install script symlinks each of the 26 skills there.

<Steps>
  <Step title="Clone the repository to a stable location">
    Clone once to `~/.claude/skills-sources/` so future `git pull` updates do not move the symlink targets.

    ```bash
    git clone https://github.com/skydoves/compose-performance-skills.git \
      ~/.claude/skills-sources/compose-performance-skills
    ```
  </Step>
  <Step title="Run the install script">
    The script creates `~/.claude/skills/` if it does not exist and symlinks each skill folder under its slug.

    ```bash
    ~/.claude/skills-sources/compose-performance-skills/scripts/install-skills.sh
    ```

    The script is idempotent — run it again after a `git pull` to pick up new skills without creating duplicate symlinks.
  </Step>
  <Step title="Restart Claude Code">
    Quit and reopen Claude Code. The skills are now active. No further configuration is required.
  </Step>
  <Step title="Describe a Compose performance symptom">
    Type a prompt that mentions a Compose performance symptom. Claude Code matches the trigger vocabulary in the skill frontmatter and loads the relevant `SKILL.md` automatically.

    ```
    My LazyColumn drops frames during fast scroll. Composables show in the
    layout inspector but recomposition counts spike on every scroll tick.
    Diagnose the stability issue and fix it.
    ```

    Claude Code matches `dropped frames`, `recomposition counts`, and `LazyColumn` against skill frontmatter and loads `stability/diagnosing-compose-stability/SKILL.md`, which chains into `stability/stabilizing-compose-types/SKILL.md` for the fix.
  </Step>
</Steps>

  </Tab>
  <Tab title="Android Studio / Gemini">

Android Studio Agent mode and Gemini discover skills at runtime by scanning project-local directories. Use the same install script, targeting your project's agent skills directory.

<Steps>
  <Step title="Navigate to your Android project root">
    Open a terminal in the root of the Android project where you want the skills available.
  </Step>
  <Step title="Clone the repository into your project">
    Clone into a hidden source directory at the project root. Add it to `.gitignore` so it is not committed.

    ```bash
    git clone https://github.com/skydoves/compose-performance-skills.git \
      .compose-performance-skills-source
    ```

    ```bash
    echo ".compose-performance-skills-source/" >> .gitignore
    ```
  </Step>
  <Step title="Run the install script targeting the agent skills directory">
    Pass the target directory as the first argument. The script symlinks each skill under `.agent/skills/<slug>/`.

    ```bash
    ./.compose-performance-skills-source/scripts/install-skills.sh .agent/skills
    ```
  </Step>
  <Step title="Verify the agent picks up the skills">
    Open Android Studio and start an Agent mode session. Describe a Compose performance symptom. The agent matches the trigger vocabulary in each skill's frontmatter against your prompt and loads the corresponding `SKILL.md`.

    <Note>
      End-to-end behavior with Android Studio Agent mode and Gemini has not been independently verified by the author. If you encounter issues, report them at the [issue tracker](https://github.com/skydoves/compose-performance-skills/issues).
    </Note>
  </Step>
</Steps>

  </Tab>
  <Tab title="Claude.ai / Claude API">

For Claude.ai workspaces and direct Anthropic API integrations, the skills are self-contained and require no install script.

<Steps>
  <Step title="Locate the relevant SKILL.md files">
    Browse the repository at [github.com/skydoves/compose-performance-skills](https://github.com/skydoves/compose-performance-skills) or clone it locally. Each skill lives at `<category>/<slug>/SKILL.md`.

    ```bash
    git clone https://github.com/skydoves/compose-performance-skills.git
    ```
  </Step>
  <Step title="Upload or reference the skill files">
    **Claude.ai workspaces**: Upload the relevant `SKILL.md` files as Agent Skill attachments in your workspace settings.

    **Anthropic API**: Reference skills inline in the system prompt, or load them through the Agent Skills file API. Attach the `references/` files cited in the skill body alongside the `SKILL.md`.
  </Step>
  <Step title="Describe a Compose performance symptom">
    Start a conversation and describe the symptom. The agent reads the loaded skill and follows its workflow.

    <Tip>
      You can load multiple skills at once. The skills cross-reference each other — a diagnostic skill names the fix skills it hands off to, so the agent can chain them without guessing.
    </Tip>
  </Step>
</Steps>

## Uninstalling

To remove the symlinks created by the install script, run it with the `--uninstall` flag. Pass the same custom target directory if you used one.

```bash
# Default target (~/.claude/skills)
~/.claude/skills-sources/compose-performance-skills/scripts/install-skills.sh --uninstall

# Custom target
./scripts/install-skills.sh --uninstall .agent/skills
```

The script removes only the symlinks it created — it does not touch symlinks from other skill libraries.

## Updating the skills

Pull the latest changes from the repository, then run the install script again. It is idempotent and skips slugs that already have a symlink in place.

```bash
cd ~/.claude/skills-sources/compose-performance-skills
git pull

# Re-run to link any newly added skills
./scripts/install-skills.sh
```
