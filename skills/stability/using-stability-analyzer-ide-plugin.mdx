---
title: "Using the Stability Analyzer IDE Plugin for Live Feedback"
description: "Install the compose-stability-analyzer IntelliJ plugin, read gutter colors and hover docs, tune inline hints, and act on the UnstableComposable inspection."
---

The Compose Compiler reports tell you about stability after a build completes. The `skydoves/compose-stability-analyzer` IntelliJ / Android Studio plugin tells you while you edit. Colored gutter icons mark every `@Composable` with a stability verdict, hover documentation renders the per-parameter table with reasons, inline hints render colored badges next to each parameter type, and the `UnstableComposable` weak-warning inspection surfaces actionable problems with quick fixes. The plugin uses the Kotlin K2 Analysis API to evaluate the same stability rules as the compiler — so editor feedback agrees with what [Diagnosing Compose Stability](/skills/stability/diagnosing-compose-stability) would later show in the compiler reports.

## When to use this skill

- You want real-time stability feedback while editing a Composable file, without running a Gradle build
- A reviewer is reading a PR locally and wants the inline signal next to each composable
- A new team member wants to see at a glance which composables in a file are non-skippable
- The editor flagged a function with a weak warning called `UnstableComposable` and you want to understand it
- You need to decide whether to suppress a warning or fix the underlying type

## Prerequisites

- Android Studio **2024.2+** or IntelliJ IDEA **2024.2+** (build range 242 through 261, covering 2024.2 through the 2026.1 EAP)
- Kotlin **2.3.0** in the IDE plugin runtime — required by the K2 Analysis API the plugin relies on
- `org.jetbrains.kotlin.plugin.compose` already wired in the project
- The JetBrains Marketplace listing is pending — install from disk during development

## Workflow

<Steps>
  <Step title="Build the plugin from source">
    Clone the repository and build the IDEA module.

    ```bash
    git clone https://github.com/skydoves/compose-stability-analyzer
    cd compose-stability-analyzer
    ./gradlew :compose-stability-analyzer-idea:buildPlugin
    ```

    The packaged plugin lands at `compose-stability-analyzer-idea/build/distributions/*.zip`.
  </Step>

  <Step title="Install from disk">
    In the IDE, open **Settings → Plugins → ⚙ → Install Plugin from Disk…** and pick the ZIP. Restart the IDE when prompted.

    <Warning>
      Do **not** look for the plugin in the JetBrains Marketplace yet — the listing is under review and is not available there.
    </Warning>
  </Step>

  <Step title="Confirm the settings panel">
    Open **Settings → Tools → Compose Stability Analyzer**. Verify `isStabilityCheckEnabled` is on. The full set of toggles:

    | Key | Default | Effect |
    |---|---|---|
    | `isStabilityCheckEnabled` | `true` | Master switch for all editor features |
    | `isStrongSkippingEnabled` | `true` | Match Compose Compiler 2.0.20+ behavior |
    | `showGutterIcons` | `true` | Colored icon beside each `@Composable` |
    | `showGutterIconsOnlyForUnskippable` | `false` | Hide green/gray icons; show only problems |
    | `showGutterIconsInTests` | `false` | Render gutter icons inside test source roots |
    | `showWarnings` | `true` | Enable the `UnstableComposable` inspection |
    | `showInlineHints` | `true` | Colored parameter-type badges inline |
    | `showOnlyUnstableHints` | `false` | Suppress inline badges for stable parameters |
    | `stabilityConfigurationPath` | `""` | Path to the project's `stability_config.conf` |
    | `isHeatmapEnabled` | `true` | Allow the live recomposition heatmap |

    Point `stabilityConfigurationPath` at the project's `stability_config.conf` so the in-editor classification matches the compiler's.
  </Step>

  <Step title="Read the gutter colors">
    Open any Kotlin file with `@Composable` functions. A colored icon appears beside each one.

    | Color | Hex | Meaning |
    |---|---|---|
    | Green | `#5FB865` | Skippable; every parameter stable |
    | Red | `#E8684A` | At least one unstable parameter; not skippable |
    | Yellow | `#F0C674` | At least one runtime-stability parameter |
    | Gray | `#808080` | No parameters; trivially skippable |

    A red or yellow gutter on a hot-path composable is a diagnostic. Confirm with the compiler reports, then fix via [Stabilizing Compose Types](/skills/stability/stabilizing-compose-types).
  </Step>

  <Step title="Hover for the parameter table">
    Hover any composable name. The popup (served by `StabilityDocumentationProvider`) shows the skippability badge, a per-parameter stability table, and the suggested fix when the analyzer can name one (e.g. "wrap `List<Item>` in `ImmutableList`"). Use this for triage before opening the full compiler reports.
  </Step>

  <Step title="Tune inline hints for dense files">
    `StabilityInlayHintsProvider` renders a colored badge next to each parameter type. On a dense file this can be noisy. Enable `showOnlyUnstableHints` so badges disappear for stable parameters and only the actionable ones remain. For an even quieter file, enable `showGutterIconsOnlyForUnskippable` so the gutter only shows red/yellow.
  </Step>

  <Step title="Triage the UnstableComposable inspection">
    `StabilityInspection` (short name `UnstableComposable`, group `Compose`, severity weak warning) flags non-skippable composables with at least one unstable parameter. It runs on the fly and under **Code → Inspect Code…**

    The inspection is intentionally a weak warning — skippability is a diagnostic, not a build-blocking error. It will not turn the file red, will not block a commit, and will not fail CI on its own. Treat it as a backlog signal for hot-path composables.
  </Step>

  <Step title="Apply quick fixes only when justified">
    Alt+Enter on a flagged composable offers two suppressions:

    - `@Suppress("NonSkippableComposable")` — added by `AddSuppressAnnotationFix`. Use on a debug overlay or composable that is never on a hot path.
    - `@Suppress("ParamsComparedByRef")` — narrows the scope: silences the warning when a parameter is intentionally compared by reference.

    Apply suppressions on the **specific function only**. Never push them to file scope.
  </Step>

  <Step title="Add @TraceRecomposition to hot composables">
    `AddTraceRecompositionIntention` is available via Alt+Enter on any `@Composable`. It inserts the `@TraceRecomposition` annotation — the runtime instrumentation entry point used by the live heatmap and by the runtime tracing skill. Use it on suspected hot composables once the gutter has identified them.
  </Step>
</Steps>

## Patterns

### Red gutter on a hot path is actionable even when the build is green

<Tabs>
  <Tab title="Wrong">
    ```text
    PriceTicker has a red gutter icon.
    Build is green, the team ships.
    Jank reports arrive two days later.
    ```
    The inspection is a weak warning by design and does not block the build. The gutter is the only signal until the developer opens the compiler reports.
  </Tab>
  <Tab title="Right">
    ```text
    PriceTicker has a red gutter icon.
    Hover -> "price: Price unstable (var price)".
    Confirm with compiler reports from diagnosing-compose-stability.
    Stabilize Price (val + immutable fields) per stabilizing-compose-types.
    Gutter turns green; confirm with the live heatmap.
    ```
  </Tab>
</Tabs>

### Suppress on the function, never on the file

<Tabs>
  <Tab title="Wrong">
    ```kotlin
    @file:Suppress("UnstableComposable")

    package com.example.feature.cart
    ```
    File-level suppression hides every future regression in that file. New unstable composables added later will never surface in the inspection.
  </Tab>
  <Tab title="Right">
    ```kotlin
    // Narrow, justified suppression on a single function
    @Suppress("NonSkippableComposable")
    @Composable
    fun DebugOverlay(state: MutableState<DebugInfo>) {
        // dev-only overlay; never on hot path; intentional MutableState parameter.
    }
    ```
  </Tab>
</Tabs>

### Silence noise via settings, not by disabling the inspection

<Tabs>
  <Tab title="Wrong">
    ```text
    Settings → Editor → Inspections → Compose → Unstable Composable → off
    ```
    Turning the inspection off globally silences real regressions across the whole codebase.
  </Tab>
  <Tab title="Right">
    ```text
    Settings → Tools → Compose Stability Analyzer:
      showGutterIconsOnlyForUnskippable = true
      showOnlyUnstableHints = true
    ```
    The inspection stays on; only the visual noise is reduced.
  </Tab>
</Tabs>

### Align the in-editor classifier with the compiler

```kotlin
// RIGHT — in Settings → Tools → Compose Stability Analyzer
isStrongSkippingEnabled = true
stabilityConfigurationPath = "<repo-root>/stability_config.conf"
```

The compiler runs with Strong Skipping on by default in Kotlin 2.0.20+ and consults the same `stability_config.conf`. Matching both settings means the gutter color you see agrees with what the compiler emits.

## Mandatory rules

- **MUST** treat a red or yellow gutter on any composable inside a hot path as actionable — confirm with compiler reports, then fix
- **MUST** point `stabilityConfigurationPath` at the same `stability_config.conf` the Compose compiler reads
- **MUST NOT** apply file-level `@file:Suppress("UnstableComposable")` — suppress on the specific function with a comment explaining why
- **MUST NOT** disable the inspection globally to silence noise — tune `showGutterIconsOnlyForUnskippable` and `showOnlyUnstableHints` instead
- **MUST NOT** chase 100% green gutters — skippability is a diagnostic, not a KPI; cold-path composables may legitimately be red
- **PREFERRED:** keep `isStrongSkippingEnabled = true` so the analyzer's classification matches the compiler's runtime behavior on Kotlin 2.0.20+
- **PREFERRED:** install from disk during development — the JetBrains Marketplace listing is pending

## Verification

- [ ] Plugin installed from disk via **Settings → Plugins → ⚙ → Install Plugin from Disk…**; IDE restarted
- [ ] **Settings → Tools → Compose Stability Analyzer** is present and `isStabilityCheckEnabled` is on
- [ ] Opening a Kotlin file with `@Composable` declarations shows a colored gutter icon beside every one of them
- [ ] Hovering a composable name shows the parameter-stability table popup
- [ ] Inline parameter-type hints render next to each parameter; toggling `showOnlyUnstableHints` hides hints for stable parameters
- [ ] The `UnstableComposable` inspection (group `Compose`, weak warning) flags at least one known-unstable composable in the open file
- [ ] Alt+Enter on a flagged composable offers `@Suppress("NonSkippableComposable")` and `@Suppress("ParamsComparedByRef")` quick fixes
- [ ] Alt+Enter on any `@Composable` offers the **Add @TraceRecomposition** intention
- [ ] `stabilityConfigurationPath` points at the same `stability_config.conf` used by the Compose Compiler in the project's Gradle config
